# HabitCircle 데스크톱 웹 버전 아키텍처 기획서

**일시:** 2026-03-27
**참여 역할:** 시스템 아키텍트, UX/UI 디자이너, 프론트엔드 아키텍트, 프로덕트 매니저

---

## 1. 현재 상태 분석

### 현재 레이아웃 제약
- 모든 보호 페이지: `max-w-lg (512px)` + `mx-auto` → 데스크톱에서 화면 중앙에 좁은 모바일 뷰
- 네비게이션: 하단 플로팅 바 (`fixed bottom-4`) → 데스크톱에서 부자연스러움
- 반응형 클래스: `sm:text-5xl` 1개만 사용 → 사실상 반응형 미구현
- CSS 변수 기반 테마: 수정 없이 확장 가능

### 데스크톱 전환 시 변경 필요 지점
| 영역 | 현재 | 데스크톱 목표 |
|------|------|-------------|
| 콘텐츠 폭 | max-w-lg (512px) | 사이드바 + max-w-4xl (896px) |
| 네비게이션 | 하단 플로팅 바 | 좌측 사이드바 |
| 레이아웃 | 1컬럼 | 대시보드: 2~3컬럼 그리드 |
| 습관 카드 | 세로 목록 | 그리드 카드 |
| 서클 진행도 | 가로 스크롤 테이블 | 넓은 그리드 |

## 2. 아키텍처 결정: 반응형 vs 별도 라우트

### 토론 요약

**옵션 A: 반응형 (Responsive) — 권장**
- 같은 컴포넌트에 `md:`, `lg:` 브레이크포인트 추가
- 장점: 코드 중복 없음, 유지보수 1벌, SEO 동일 URL
- 단점: 모바일과 데스크톱 UX 타협 가능성

**옵션 B: 별도 라우트 (Parallel Routes)**
- `/desktop/dashboard`, `/mobile/dashboard` 분리
- 장점: 각 플랫폼에 최적화된 UX
- 단점: 코드 2벌, API 공유해도 뷰 로직 중복

**결정: 옵션 A (반응형)** — 이유:
1. 현재 팀 규모(1인)에서 2벌 유지보수는 비현실적
2. Tailwind v4의 반응형 시스템이 충분히 강력
3. 핵심 데이터/로직은 동일, 레이아웃만 다름
4. 세계적 트렌드: Notion, Linear, Todoist 모두 반응형

## 3. 레이아웃 설계

### 3-1. 브레이크포인트 전략

```
< 768px (md)  → 모바일: 현재 그대로 (하단 네비게이션)
>= 768px (md) → 태블릿: 사이드바 축소 모드 (아이콘만)
>= 1024px (lg) → 데스크톱: 사이드바 확장 모드 (아이콘 + 텍스트)
>= 1280px (xl) → 와이드: 최대 콘텐츠 폭 + 여백
```

### 3-2. 메인 레이아웃 구조

```
┌─────────────────────────────────────────────────────┐
│ (모바일: < md)                                        │
│ ┌─────────────────────────────────────────────┐      │
│ │           콘텐츠 (max-w-lg)                  │      │
│ │                                             │      │
│ └─────────────────────────────────────────────┘      │
│         ┌─────────────────────┐                      │
│         │   하단 네비게이션 바   │                      │
│         └─────────────────────┘                      │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ (데스크톱: >= md)                                      │
│ ┌──────┐ ┌──────────────────────────────────────┐    │
│ │      │ │                                      │    │
│ │ 사이드│ │        메인 콘텐츠 영역               │    │
│ │  바  │ │        (max-w-4xl)                   │    │
│ │      │ │                                      │    │
│ │ 홈   │ │  ┌──────────┐ ┌──────────────────┐  │    │
│ │ 습관  │ │  │ 습관 목록  │ │ 활동 피드 / 통계  │  │    │
│ │ 서클  │ │  │          │ │                  │  │    │
│ │ 친구  │ │  │          │ │                  │  │    │
│ │ 프로필│ │  └──────────┘ └──────────────────┘  │    │
│ │      │ │                                      │    │
│ └──────┘ └──────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

### 3-3. 사이드바 네비게이션 설계

```
┌──────────────────┐
│  ✓  HabitCircle  │  ← 로고 (lg: 텍스트, md: 아이콘만)
│                  │
│  ◉  홈           │  ← 활성 상태: bg-teal/10, text-teal
│  ○  습관          │  ← 비활성: text-warm-gray
│  ○  서클          │
│  ○  친구          │  ← 현재 모바일 네비에 없음, 추가
│                  │
│  ─────────────── │
│                  │
│  ○  프로필        │
│  ○  로그아웃      │  ← 데스크톱에서 접근성 개선
└──────────────────┘
```

**사이드바 크기:**
- `md`: 아이콘만 → `w-16` (64px)
- `lg`: 아이콘 + 텍스트 → `w-56` (224px)

## 5. 구현 계획

### 수정 대상 파일 (14개)

| # | 파일 | 변경 내용 |
|---|------|----------|
| 1 | `src/app/(main)/layout.tsx` | 사이드바 + 반응형 컨테이너 (가장 큰 변경) |
| 2 | `src/app/(main)/dashboard/page.tsx` | 2컬럼 그리드 (습관 + 피드/통계) |
| 3 | `src/app/(main)/habits/page.tsx` | 카드 그리드 |
| 4 | `src/app/(main)/habits/[id]/page.tsx` | 히트맵 + 통계 나란히 |

| 6 | `src/app/(main)/circles/page.tsx` | 카드 그리드 |
| 7 | `src/app/(main)/circles/[id]/page.tsx` | 넓은 테이블 + 멤버 사이드 |

| 9 | `src/app/(main)/circles/join/page.tsx` | 폼 폭 확장 |

| 11 | `src/app/(main)/profile/page.tsx` | 통계 그리드 |
| 12 | `src/app/(auth)/login/page.tsx` | 좌우 분할 |

| 14 | `src/app/page.tsx` | 좌우 분할 랜딩 |

### 구현 순서

**Phase 1: 레이아웃 인프라 (layout.tsx)**
- 사이드바 컴포넌트 구현
- `md:` 브레이크포인트에서 하단 네비 → 사이드바 전환
- 메인 콘텐츠 `max-w-lg` → `md:max-w-4xl` 확장

**Phase 2: 핵심 페이지 (대시보드 + 습관)**
- 대시보드 2컬럼 그리드
- 습관 목록 카드 그리드
- 습관 상세 좌우 분할

**Phase 3: 소셜 페이지 (서클 + 친구 + 프로필)**
- 서클 상세 넓은 테이블
- 친구 2컬럼
- 프로필 통계 그리드

**Phase 4: 인증 페이지 (랜딩 + 로그인 + 회원가입)**
- 좌우 분할 레이아웃
- 데스크톱 전용 장식 요소

### 핵심 원칙

1. **모바일 코드를 건드리지 않는다** — 모든 변경은 `md:` 이상 브레이크포인트에만 적용
2. **기존 CSS 변수를 그대로 사용** — 새 변수 추가 최소화
3. **각 페이지의 기능 로직은 변경하지 않음** — 레이아웃/스타일만 변경
4. **사이드바는 layout.tsx 한 곳에서 관리** — 개별 페이지에 사이드바 코드 없음

## 6. 검증 방법

1. **반응형 테스트**: Chrome DevTools에서 375px(iPhone), 768px(iPad), 1280px(Desktop), 1920px(FHD) 뷰포트 확인
2. **네비게이션 전환**: md 브레이크포인트에서 하단 바 ↔ 사이드바 전환이 매끄러운지
3. **콘텐츠 깨짐**: 모든 페이지에서 텍스트 잘림, 오버플로, 정렬 이슈 없는지
4. **빌드 확인**: `npm run build` 성공
5. **기존 모바일 UX**: 모바일 뷰에서 기존 동작이 그대로인지 회귀 테스트
